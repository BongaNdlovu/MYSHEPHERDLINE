# Data Subject Request Workflow (Draft)

## Supported request types

- Access
- Correction
- Deletion
- Objection to processing (where applicable)

## Intake

1. Request received via Information Officer contact channel
2. Verify requester identity and authority (member, guardian where applicable, or authorized representative)
3. Log request with date, type, and reference ID

## Processing

1. Locate records across Supabase tables and operational logs
2. Apply legal review for special personal information and religious-context data
3. Fulfill or lawfully refuse with written reasons

## Response timelines

Follow POPIA statutory timelines and internal SLA targets.

## Deletion caveats

Deletion may be limited where retention is required by law or legitimate ongoing pastoral/legal obligations.

## Record keeping

Retain request logs and outcomes per retention schedule.
